Description

In this lab I explored analog input on the Arduino using a potentiometer and a joystick. I measured a changing analog voltage with analogRead(), then mapped that value to LED output. Next, I read the joystick's X and Y axes and used thresholding to detect directional pushes. Finally, I built a Whack-A-Mole style game where one LED is the target and the joystick direction is used to hit the correct LED and a time is given at the end.

What it Does / How to Use It

  • Part 1: Turn the potentiometer knob and watch the raw ADC value + voltage print in Serial Monitor.
  • Part 2: Turn the potentiometer knob to control how many LEDs turn on (like a bar graph).
  • Part 3: Move the joystick and watch the X/Y values change in Serial Monitor. Press the joystick to change SW.
  • Part 4: Move the joystick up/down/left/right to light the matching LED in the plus layout.
  • Part 5: When the target LED lights up, move the joystick in the matching direction to “hit” it. A time is shown at the end.

How it Works (For an Informed Reader)

The potentiometer and joystick axes are read through the Arduino’s ADC (0–1023). The potentiometer is a voltage divider: the wiper outputs a voltage between 0V and 5V depending on knob position. The joystick VRx/VRy behave the same way but are mechanically controlled by the stick position. The LED behaviors are driven with digital output pins. For the bar graph, the ADC reading is mapped to an integer count of LEDs to illuminate. For joystick direction, thresholds and a deadzone are used to convert analog values into discrete directions.

Math / Calculations

Potentiometer voltage conversion

The Arduino ADC returns 0–1023 for 0–5V (assuming default 5V reference). I converted raw readings to voltage using: 

voltage = raw * 5.0 / 1023.0

Mapping potentiometer range to LED count

To display a 5-LED bar, I mapped the potentiometer range to the number of LEDs to turn on: 

ledOn = map(potValue, 0, 1023, 0, 5)

Example: If potValue is about 512, ledOn is about 2–3 LEDs depending on rounding.

// analog pin where the potentiometer is connected
const int potentiometer = A0;

void setup() {
  Serial.begin(9600);
}

void loop() {
  // reads the potentiometer
  int raw = analogRead(potentiometer);

  // convert to volts
  float voltage = raw * 5.0 / 1023.0;

  // print everything and delay
  Serial.print("raw = ");
  Serial.print(raw);
  Serial.print(" V = ");
  Serial.println(voltage);

  delay(500);
}

Part 2 — Map Potentiometer Value to LED Output

Procedure

  1. Wired 5 LEDs each with a 220Ω resistor to digital pins 2–6.
  2. Read the potentiometer value (0–1023) from A0.
  3. Used map() to convert the potentiometer value into how many LEDs should be on (0–5).
  4. Turned on that many LEDs from the array so the “bar” grows/shrinks when the knob turns.

// LED pins (left to right)
int pin_LEDarray[] = {2, 3, 4, 5, 6};

// potentiometer analog pin
#define pin_potentiometer A0

void setup() {
  // set each LED pin as an output
  for (int i = 0; i < 5; i++) {
    pinMode(pin_LEDarray[i], OUTPUT);
  }
}

void loop() {
  // start by turning all LEDs off so we can redraw the bar cleanly
  for (int i = 0; i < 5; i++) {
    digitalWrite(pin_LEDarray[i], LOW);
  }

  // read potentiometer (0 to 1023)
  int potValue = analogRead(pin_potentiometer);

  // convert pot range to number of LEDs on (0 to 5)
  int ledOn = map(potValue, 0, 1023, 0, 5);

  // turn on the first ledOn LEDs
  for (int i = 0; i < ledOn; i++) {
    digitalWrite(pin_LEDarray[i], HIGH);
  }
}

Part 3 — Joystick Analog Read (X/Y) + Button

Procedure

  1. Connected joystick VCC to 5V and GND to GND.
  2. Connected VRx to A0 and VRy to A1.
  3. Connected SW to digital pin 2 and read it with digitalRead().
  4. Printed X, Y, and SW values to Serial Monitor.

// VRx
const int xPin = A0;
// VRy
const int yPin = A1;
// SW
const int swPin = 2;

void setup() {
  Serial.begin(9600);
  pinMode(swPin, INPUT);
}

void loop() {
  // 0 to 1023
  int xVal = analogRead(xPin);

  // 0 to 1023
  int yVal = analogRead(yPin);

  // HIGH when pressed, LOW otherwise
  int swVal = digitalRead(swPin);

  Serial.print("X = ");
  Serial.print(xVal);
  Serial.print("  Y = ");
  Serial.print(yVal);
  Serial.print("  SW = ");
  Serial.println(swVal);

  delay(50);
}

Conclusion

This lab showed how analog sensors produce a range of values and how to turn those readings into actions in code. The potentiometer was a clear example of reading a variable voltage, while the joystick required thresholding and a deadzone to reliably detect direction. Combining joystick input with a plus layout of LEDs made the mapping easy to understand, and the final game tied analog input, logic, and output together in a single project.
